Personal Reflection

Journey

Before this course, I viewed programming as a linear process of writing instructions for a computer. Data Structures and Algorithms (DSA) fundamentally reshaped that perspective, teaching me to see code not just in terms of syntax, but through the lens of efficiency, structure, and elegance. This journey was as much about training my mind to think logically as it was about mastering technical concepts.

Efficiency Structure Elegance

Technical and Personal Growth

Technically, I transitioned from a coder who could make things work to a developer who thinks about how they work. Learning about time and space complexity (Big-O notation) was a paradigm shift. I moved from asking "Does this code run?" to "How well does this code run with a million data points?" Implementing core data structures from the basic arrays and linked lists to the complex trees and graphs. It gave me a toolkit for organizing data intelligently. Similarly, mastering algorithms like Bubble Sort versus Quicksort, or DFS versus BFS, taught me that there is no single "best" solution, only the most optimal one for a given context.
